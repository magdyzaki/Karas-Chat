# الشات من الصفر — خطوة بخطوة

## الخطوة 1: إنشاء مشروع جديد على GitHub

1. افتح المتصفح وادخل على: **https://github.com**
2. إذا لم يكن لديك حساب: اضغط **Sign up** وأنشئ حساباً (بريد، كلمة مرور، اسم مستخدم).
3. بعد تسجيل الدخول اضغط **+** أعلى الصفحة (أو من القائمة) واختر **New repository**.
4. املأ:
   - **Repository name:** اسم للمشروع، مثلاً `my-chat-app` (بدون مسافات).
   - **Description:** اختياري، مثلاً "شات ويب".
   - اختر **Public**.
   - **لا** تضف README أو .gitignore الآن (المشروع موجود عندك فعلاً).
5. اضغط **Create repository**.

بعدها تظهر لك صفحة فيها تعليمات. **اترك الصفحة مفتوحة** — سنستخدم رابط المستودع في الخطوة التالية.

---

## الخطوة 2: تثبيت Git على جهازك (إن لم يكن مثبتاً)

- **Windows:** حمّل من https://git-scm.com/download/win وثبّت، ثم أعد فتح الطرفية (PowerShell أو CMD).
- للتأكد من التثبيت اكتب في الطرفية:
  ```bash
  git --version
  ```
  إذا ظهر رقم إصدار فـ Git جاهز.

---

## الخطوة 3: رفع مجلد المشروع إلى GitHub من جهازك

1. افتح **PowerShell** أو **Command Prompt**.
2. ادخل لمجلد المشروع الكبير (الذي فيه مجلد `chat-app`):
   ```bash
   cd D:\programs\Smart_CRM_Final_Arabic
   ```
   (غيّر المسار إذا كان مشروعك في مكان آخر.)
3. إذا كان هذا المجلد **لم** يُربط بـ Git من قبل، نفّذ الأوامر التالية **بالترتيب** (واستبدل `YOUR_USERNAME` و `YOUR_REPO` باسم مستخدمك على GitHub واسم المستودع):

   ```bash
   git init
   git add .
   git commit -m "إضافة مشروع الشات"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
   git push -u origin main
   ```

   **مثال:** لو اسم المستخدم `ahmed` واسم المستودع `my-chat-app`:
   ```bash
   git remote add origin https://github.com/ahmed/my-chat-app.git
   ```

4. عند أول `git push` قد يطلب منك GitHub **اسم المستخدم وكلمة المرور**. استخدم:
   - اسم المستخدم = اسمك على GitHub
   - كلمة المرور = **Personal Access Token** (وليس كلمة مرور الحساب العادية).  
   لإنشاء التوكن: GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Generate new token**، اختر صلاحية **repo**، انسخ التوكن واستخدمه مكان كلمة المرور عند الطلب.

بعد نجاح `git push` ادخل على صفحة المستودع على GitHub — هتشوف كل الملفات مرفوعة. **كده خلصت الجزء الخاص بـ GitHub.**

## الخطوة 4: نشر الباكند على Koyeb

1. ادخل على **https://www.koyeb.com** وأنشئ حساباً (أو سجّل دخول).
2. من لوحة Koyeb اضغط **Create App** أو **Create Web Service**.
3. اختر **GitHub** كمصدر، واسمح لـ Koyeb بالوصول لمستودعاتك واختر المستودع اللي رفعت فيه المشروع (مثلاً `my-chat-app`).
4. في الإعدادات:
   - **Root directory:** اكتب `chat-app/backend`
   - **Build command:** `npm install`
   - **Run command:** `npm start`
5. في **Environment variables** اضف:
   - `JWT_SECRET` = أي نص سري طويل (مثلاً: `my-super-secret-key-12345`)
   - `CORS_ORIGIN` = اتركه فارغاً الآن
6. اضغط Deploy. بعد ما يخلص، هيديك رابط مثل: `https://my-chat-app-xxxx.koyeb.app`  
   **انسخ هذا الرابط** — ده رابط الباكند.

---

## الخطوة 5: نشر الواجهة على Vercel

1. ادخل على **https://vercel.com** وسجّل دخول (ممكن عبر GitHub).
2. اضغط **Add New…** ثم **Project** واختر نفس المستودع من GitHub.
3. في إعدادات المشروع:
   - **Root Directory:** اضغط **Edit** واختر **`chat-app/frontend`** فقط.
   - في **Environment Variables** اضف:
     - الاسم: `VITE_API_URL`
     - القيمة: **رابط الباكند من Koyeb** (مثل `https://my-chat-app-xxxx.koyeb.app`) **بدون** `/` في الآخر.
4. اضغط **Deploy**. بعد ما يخلص هيديك رابط مثل: `https://my-chat-app-xxxx.vercel.app`  
   **ده رابط الشات اللي تشاركه مع الناس.**

---

## الخطوة 6: ربط الباكند بالواجهة (CORS)

1. ارجع لـ **Koyeb** → مشروع الباكند → **Settings** أو **Environment**.
2. عدّل المتغير `CORS_ORIGIN` وضعه = **رابط Vercel بالضبط** (مثل `https://my-chat-app-xxxx.vercel.app`).
3. احفظ وأعد النشر لو طلب منك.

---

## خلاص

- **GitHub:** فيه كود المشروع.
- **Koyeb:** يشغّل الباكند.
- **Vercel:** يشغّل الواجهة.

**الرابط اللي تعطيه لأي حد عشان يدخل الشات:** رابط Vercel.

لو وقعت في أي خطوة (مثلاً خطأ في الأمر أو في الإعدادات)، اكتب لي رقم الخطوة والرسالة اللي تظهر عندك وأقولك نعمل إيه.
