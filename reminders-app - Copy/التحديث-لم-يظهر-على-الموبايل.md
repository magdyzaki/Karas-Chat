# التحديث لم يظهر على الموبايل — ماذا تفعل؟

---

## 1) تأكد أنك تشوف النسخة الجديدة

في الصفحة الرئيسية (بعد تسجيل الدخول) تحت نص «مرحباً...» يفترض أن يظهر سطر صغير:

**«نسخة واجهة: 2»**

- **لو ظهر** = التحديث محمّل؛ إذن المشكلة كانت من الكاش وتم حلها.
- **لو لم يظهر** = المتصفح ما زال يعرض نسخة قديمة (كاش) أو الملفات المرفوعة على GitHub قديمة.

---

## 2) مسح الكاش على الموبايل

### Chrome (أندرويد)
1. افتح **reminders-frontend.vercel.app**
2. اضغط أيقونة **القفل** أو **ⓘ** بجانب الرابط
3. اختر **إعدادات الموقع** أو **Site settings**
4. اختر **مسح البيانات المحفوظة** أو **Clear & reset** ثم احذف البيانات
5. أعد فتح الرابط

أو: افتح الرابط في **نافذة خاصة (Incognito)** — إن ظهرت «نسخة واجهة: 2» فالسبب كان الكاش.

### Safari (آيفون)
1. **الإعدادات** → **Safari** → **مسح السجل والبيانات**
2. أو افتح الرابط في **وضع التصفح الخاص**

---

## 3) التأكد أن GitHub فيه الكود الجديد

1. ادخل **github.com/magdyzaki/reminders-frontend**
2. افتح الملف **src/RemindersList.jsx**
3. ابحث عن النص **«نسخة واجهة: 2»** أو **«التنبيه مع الشاشة مطفية»**
4. **لو غير موجود** = الملف المرفوع قديم؛ انسخ من جهازك محتوى الملفات الحالية (من مجلد `reminders-app/frontend/src/`) والصقها في GitHub ثم Commit

---

## 4) بعد رفع التحديث على GitHub

- انتظر 1–2 دقيقة ثم على Vercel تأكد أن آخر **Deployment** من commit الجديد وحالته **Ready** و **Current**.
- على الموبايل: **امسح بيانات الموقع** (أو افتح في نافذة خاصة) ثم افتح الرابط مرة أخرى وتأكد من ظهور «نسخة واجهة: 2».
